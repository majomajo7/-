# GPT Audio Splitter v2

오디오 파일을 사용자가 지정한 개수만큼 같은 길이로 나누고, GPT 전사에 적합한 MP3 파일로 변환하는 웹앱입니다. v2는 큰 파일을 5MB 단위로 나눠 업로드하여 무료 서버에서 발생하던 HTTP 502 가능성을 줄였습니다.

## 동작 방식

- 입력: MP3, M4A, WAV, WEBM, OGG, OPUS, AAC, FLAC, MP4, MOV 등
- 사용자가 입력하는 값: 분할 개수 하나(2~30)
- 출력: 같은 길이의 `MP3 / mono / 16 kHz / 48 kbps`
- 파일명: `원본명_part_01_of_04.mp3` 형태
- 결과: 모든 조각과 사용안내를 하나의 ZIP으로 다운로드
- 큰 파일 업로드: 브라우저가 5MB 단위로 순차 전송하고 실패 시 자동 재시도
- 개인정보: 처리용 임시 파일은 다운로드 응답이 끝난 뒤 자동 삭제

48 kbps 모노 MP3는 약 **시간당 21.6MB**입니다. 따라서 2시간짜리 녹음을 2개로 나누면 파일당 약 21.6MB가 되어, 25MB 제한을 적용하는 전사 API에도 비교적 안전합니다.

## 가장 쉬운 온라인 배포: Render

1. 이 폴더의 파일을 GitHub의 새 저장소에 업로드합니다.
2. Render에 로그인한 뒤 **New > Blueprint**를 선택합니다.
3. GitHub 저장소를 연결합니다. 루트의 `render.yaml`이 자동으로 인식됩니다.
4. `ACCESS_KEY`에 본인만 사용할 접속 암호를 입력합니다. 공개 사용이라면 빈 값으로 두어도 됩니다.
5. Deploy를 실행합니다.
6. 배포가 끝나면 `https://...onrender.com` 형태의 주소가 생성됩니다. 이 주소는 PC, 휴대폰 등 어디서든 사용할 수 있습니다.

### Render 무료 서비스 참고

- 일정 시간 사용하지 않으면 서비스가 잠들기 때문에 첫 접속 시 잠시 늦을 수 있습니다.
- 임시 파일만 사용하므로 Persistent Disk는 필요하지 않습니다.
- 매우 큰 오디오나 동시 사용자가 많으면 유료 인스턴스가 더 안정적입니다.

## 다른 Docker 호스팅에 배포

Docker를 지원하는 Railway, Fly.io, Google Cloud Run, AWS App Runner 등의 서비스에도 그대로 배포할 수 있습니다. 컨테이너는 환경변수 `PORT`가 있으면 해당 포트를 사용합니다.

### 환경변수

| 이름 | 기본값 | 설명 |
|---|---:|---|
| `MAX_UPLOAD_MB` | `500` | 한 파일의 최대 업로드 크기(MB) |
| `MAX_SPLITS` | `30` | 최대 분할 개수 |
| `ACCESS_KEY` | 빈 값 | 설정하면 웹 화면에서 접속 암호를 요구 |
| `UPLOAD_CHUNK_MB` | `5` | 큰 파일을 전송할 때 사용하는 업로드 조각 크기(MB) |

## 로컬 테스트(선택 사항)

Docker가 설치되어 있다면 다음 명령으로 확인할 수 있습니다.

```bash
docker build -t gpt-audio-splitter .
docker run --rm -p 10000:10000 -e ACCESS_KEY=my-password gpt-audio-splitter
```

브라우저에서 `http://localhost:10000`을 엽니다.
