# 기존 GitHub 저장소 업데이트 방법

1. 이 ZIP 파일을 압축 해제합니다.
2. GitHub의 기존 오디오 분할 저장소를 엽니다.
3. `Add file` → `Upload files`를 누릅니다.
4. 압축을 푼 폴더 **안의 항목 전체**를 선택해서 업로드 영역에 끌어다 놓습니다.
5. 목록에 `app/main.py`, `app/static/app.js`, `render.yaml`, `requirements.txt`가 보이는지 확인합니다.
6. 아래의 `Commit changes`를 누릅니다.
7. Render가 자동으로 다시 배포할 때까지 기다립니다.
8. Render가 `Live`가 되면 웹페이지에서 `Ctrl + F5`를 눌러 새 버전을 불러옵니다.
9. 페이지 상단에 `AUDIO UTILITY · LARGE FILE MODE`가 보이면 업데이트 완료입니다.

## v2에서 변경된 점

- 파일을 한 번에 보내지 않고 5MB 단위로 나눠 업로드
- 각 업로드 조각 실패 시 최대 3회 자동 재시도
- 같은 조각이 중복 전송되어도 안전하게 처리
- 서버에서 원본 파일을 중복 저장하지 않도록 변경
- 분할이 끝나면 원본을 먼저 삭제해 임시 디스크 사용량 절감
- MP3를 ZIP에 다시 압축하지 않아 처리 속도와 메모리 사용량 개선
